# MeetingEZ 使用指南

## 启动

```bash
source venv/bin/activate
pip install -r requirements.txt
python run.py
```

打开：`http://localhost:5090`

## 环境变量

至少需要：

- `OPENAI_API_KEY`

可选：

- `ACCESS_CODE`
  配置后启用登录页；为空则直接进入应用。
- `TRANSLATION_MODEL`
  默认 `gpt-5.4-mini-2026-03-17`
- `TRANSLATION_REASONING_EFFORT`
  可选。仅对支持 reasoning 的翻译模型生效；默认 `low`。

参考：[`env.example`](/home/wengzhiwen/meetingEZ/env.example)

## 页面操作

### 1. 登录

如果配置了 `ACCESS_CODE`：

- 访问首页会先跳到登录页
- 输入访问码后进入控制台首页

### 2. 选择入口

首页控制台提供两条流程：

- `快速转写`
  不关联项目，直接进入实时页
- `项目模式`
  先选项目、建立会议，再进入实时页

如果从项目模式进入：

- 会先创建会议目录
- 同时写入 `_meeting.json`
- 实时页会自动带上项目、会议、语言画像信息

### 3. 打开设置

点击底部工具栏中的 `设`：

- 测试后端 OpenAI 连接
- 查看当前连接状态
- 查看当前项目模式 / 快速模式
- 选择音频输入源
- 选择麦克风设备
- 配置主要语言 / 第二语言
- 配置语言模式
- 在项目模式下选择项目增强来源
- 调整字体大小

### 4. 开始会议

点击底部工具栏最右侧按钮 `开始`。

当前按钮是单一切换按钮：

- 未开始时：显示 `开始`
- 会议进行中：显示 `结束`

### 5. 字幕显示

转写过程中：

- 检测到讲话时，会先显示“正在识别...”
- 收到 `delta` 后实时更新
- 收到 `completed` 后写入最终字幕

如果设置了第二语言：

- 原文先进入列表
- 翻译完成后在后面插入翻译行

### 6. 其他工具

底部工具栏还包括：

- `测`
  麦克风测试
- `滚`
  自动滚动开关
- `下`
  下载 TXT 记录
- `空`
  清空当前记录

## 音频输入源

### 麦克风

适用于本机说话场景。

### 标签页音频

适用于转写浏览器中的远程会议。

操作要点：

1. 在设置里切换到“使用标签页音频”
2. 点击 `开始`
3. 浏览器弹窗里选择正确的标签页
4. 勾选“共享标签页音频”

## 当前体验说明

### 状态显示

- 页面顶部显示当前模式：
  - `项目模式`
  - `快速模式`
- 未开始时，底部计时区显示状态文字
- 会议进行中，显示累计时长
- 音量条与计时在同一工具栏内
- 连接状态仅在设置浮层内显示

### 项目模式与快速模式

- 项目模式
  会加载项目术语、背景说明、近期会议和待办摘要来增强实时转写与后置处理
- 快速模式
  不关联项目，不加载项目增强包，但仍可使用实时转写、智能修正和翻译

### 本地存储

浏览器会保存：

- 字幕记录
- 音频输入源
- 设备选择
- 主要语言 / 第二语言
- 语言模式
- 最近选择的项目
- 字体大小
- 自动滚动开关

## 导出

点击 `下` 会导出 TXT：

- 导出时间
- 条目顺序
- 每条字幕内容

## 常见问题

### 没配置 `ACCESS_CODE` 会怎样？

应用直接开放，不显示登录页。

### 为什么点击开始后浏览器弹出权限请求？

因为页面需要：

- 麦克风权限
- 或屏幕/标签页共享权限

### 翻译慢怎么排查？

看两处日志：

- 浏览器 console：`UI [perf]`
- 后端日志：`[perf]`

### 连续讲话为什么以前会卡到说完才出字？

当前实现已经改成 `semantic_vad + eagerness: high`，比旧的纯静音切段更适合长句连续讲话。
